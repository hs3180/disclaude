# Agent 服务质量周评

每周一上午 9 点分析所有聊天记录，评估 Agent 服务质量，生成对比报告。

**关联 Issue**: #1334
**Skill**: daily-quality-review

## 配置说明

**重要**: 使用前请将 `chatId` 替换为实际的飞书 Chat ID，并将 `enabled` 设为 `true`。

## 执行步骤

### 1. 获取所有聊天日志文件

```bash
ls workspace/logs/**/*.md 2>/dev/null || echo "No log files found"
```

如果 `workspace/logs/` 目录不存在或为空，跳过本次执行。

### 2. 读取聊天日志

使用 `Glob` 工具查找所有日志文件：
```
Glob workspace/logs/**/*.md
```

对于最近 7 天的日志文件，使用 `Read` 工具读取内容。

### 3. 按维度评估服务质量

对每个聊天会话，从以下 5 个维度进行评估：

#### 3.1 响应效率
- 统计每个任务的对话轮次
- 识别需要多次澄清的场景
- 关注 Agent 是否主动提供完整回答

**评分标准**: ⭐⭐⭐ (1-2轮完成) / ⭐⭐ (3-5轮) / ⭐ (多轮或需大量引导)

#### 3.2 任务完成度
- 跟踪每个任务是否达到满意结论
- 记录被放弃的任务
- 分析未完成的原因

**评分标准**: ⭐⭐⭐ (完全完成) / ⭐⭐ (基本完成) / ⭐ (未完成)

#### 3.3 用户满意度
- 统计正面信号: "谢谢", "好的", "很好", "赞"
- 统计负面信号: "不对", "错了", "应该是", "改一下"
- 识别用户沮丧信号: "又来了", "怎么还是不行"

**评分标准**: ⭐⭐⭐ (多数正面) / ⭐⭐ (混合) / ⭐ (频繁纠正)

#### 3.4 工具使用效率
- 分析工具调用次数与任务复杂度的关系
- 识别冗余或不必要的工具调用
- 评估工具选择的适当性

**评分标准**: ⭐⭐⭐ (高效精准) / ⭐⭐ (合理) / ⭐ (过多或不当)

#### 3.5 错误率
- 统计 Agent 错误次数
- 记录自修复次数
- 分析错误类型和严重程度

**评分标准**: ⭐⭐⭐ (无错误或快速自修复) / ⭐⭐ (少量错误) / ⭐ (频繁错误)

### 4. 汇总并生成报告

### 5. 发送报告

使用 `send_user_feedback` 将报告发送到配置的 chatId：

```
send_user_feedback({
  content: [报告内容],
  format: "text",
  chatId: [配置的 chatId]
})
```

## 错误处理

1. 如果日志文件读取失败，跳过该文件继续处理其他文件
2. 如果 `send_user_feedback` 失败，记录日志并重试一次
3. 如果聊天记录不足（少于 3 条有效消息），跳过该会话
4. 如果没有足够的分析数据，发送简要报告说明情况

## 验收标准

- [x] 能通过定时任务触发
- [x] 能分析 5 个质量维度
- [x] 能生成结构化评估报告
- [x] 零代码侵入（不修改任何 Agent 核心代码）
- [x] 使用 LLM 驱动分析（非硬编码算法）
- [x] 通过 send_user_feedback 发送报告
