# Agent Framework 赛马报告

每周一 9:00 分析过去一周的聊天记录，对比不同 Agent 的服务质量表现，生成评估报告。

**关联 Issue**: #1334

## 配置说明

**重要**: 使用前请将 `chatId` 替换为实际的飞书 Chat ID，并将 `enabled` 设为 `true`。

## 执行步骤

### 1. 获取所有聊天日志文件

使用 `Glob` 工具查找所有日志文件：
```
Glob workspace/logs/**/*.md
```

使用 `Bash` 列出聊天目录：
```bash
ls workspace/logs/ 2>/dev/null || echo "No log directories found"
```

如果 `workspace/logs/` 目录不存在或为空，跳过本次执行。

### 2. 读取聊天日志

对于最近 7 天的日志文件，使用 `Read` 工具读取内容。

使用 `Bash` 获取当前日期：
```bash
date -u +"%Y-%m-%dT%H:%M:%SZ"
```

### 3. 分析各聊天记录

对每个聊天目录进行以下分析：

#### 3.1 响应效率分析
- 从消息时间戳计算 TTFR (Time To First Response)
- 统计平均、P50、P90 响应时间
- 识别超过 120 秒的慢响应

#### 3.2 任务完成度分析
- 识别任务完成信号 ("✅", "完成", "done")
- 识别放弃信号 ("算了", "不用了", "never mind")
- 计算完成率

#### 3.3 用户反馈分析
- 正面信号: "谢谢", "好的", "👍", "有用"
- 负面信号: "不对", "错了", "重新来"
- 纠正模式: 用户反复纠正 Agent 输出

#### 3.4 工具使用效率分析
- 统计工具调用次数
- 识别失败的工具调用
- 检测冗余操作

#### 3.5 错误率分析
- 任务失败次数
- 超时次数
- 系统错误

#### 3.6 独特特性分析
- 各 Agent 的优势领域
- 特殊能力或知识
- 沟通风格特点

### 4. 生成对比报告

按照以下格式生成报告：

```markdown

**分析时间**: {timestamp}
**分析范围**: 过去 7 天
**分析聊天数**: {count}

---

### 📊 总览评分

| 聊天 | 响应效率 | 任务完成度 | 用户满意度 | 工具效率 | 错误率 | 综合 |
|------|----------|-----------|-----------|---------|--------|------|
| {chat1} | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ |

---

### 📈 详细分析

(包含各维度的详细数据和对比表格)

### 🌟 独特特性

(各 Agent 的定性分析)

### 🔍 关键发现

(重要发现和模式)

(优先级排序的改进建议)
```

### 5. 发送报告

使用 `send_user_feedback` 将报告发送到配置的 chatId：

```
send_user_feedback({
  content: [报告内容],
  format: "text",
  chatId: [配置的 chatId]
})
```

## 错误处理

1. 如果日志文件读取失败，跳过该文件继续处理其他文件
2. 如果 `send_user_feedback` 失败，记录日志并重试一次
3. 如果数据不足（少于 2 个有效聊天），发送简要报告说明"数据不足"
4. 如果没有检测到任何有意义的数据，发送"无异常发现"的简要报告

## 评分标准

| 维度 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ |
|------|----------|---------|--------|
| 响应效率 | Avg < 5s | Avg < 15s | Avg < 30s |
| 任务完成度 | > 95% | > 85% | > 70% |
| 用户满意度 | 几乎全部正面 | 大部分正面 | 混合反馈 |
| 错误率 | < 2% | < 5% | < 10% |
