# Agent Framework 赛马评估

每周一 09:00 分析所有聊天记录中的 Agent 表现，对比不同模型/Framework 的服务质量，生成评估报告。

**关联 Issue**: #1334

## 配置说明

**重要**: 使用前请将 `chatId` 替换为实际的飞书 Chat ID，并将 `enabled` 设为 `true`。

## 执行步骤

### 1. 获取所有聊天日志文件

```bash
ls workspace/logs/**/*.md 2>/dev/null || ls workspace/chat/*.md 2>/dev/null || echo "No log files found"
```

如果日志目录为空，跳过本次执行。

### 2. 读取聊天日志

使用 `Glob` 工具查找所有日志文件：
```
Glob workspace/logs/**/*.md
```

对于最近 7 天的日志文件，使用 `Read` 工具读取内容。

### 3. 分析 Agent 表现

使用 `agent-benchmark` skill 的分析流程，从以下维度评估：

#### 3.1 响应效率
- 用户请求到 Agent 响应的时间间隔
- 完成任务所需的对话轮次
- 一次解决率

#### 3.2 任务完成度
- 用户请求是否被完全解决
- 是否需要重试或重新表述
- 是否有后续纠正

#### 3.3 用户满意度
- 显式正面反馈（感谢、好评）
- 显式负面反馈（不满、重试）
- 隐式信号（继续对话 vs 沉默）

#### 3.4 工具使用效率
- 工具调用次数与任务复杂度的关系
- 是否存在冗余工具调用
- 工具调用成功率

#### 3.5 错误率
- 任务失败、超时、崩溃次数
- 输出错误需要纠正的次数
- 工具调用失败次数

### 4. 对比不同框架/模型

如果日志中可以识别不同的 Agent 类型或模型，进行横向对比：

| 维度 | 模型 A | 模型 B |
|------|--------|--------|
| 平均响应时间 | Xs | Ys |
| 任务完成率 | X% | Y% |
| 用户满意度 | X/5 | Y/5 |
| 错误率 | X% | Y% |

对独特特性进行定性描述，不做硬性排名。

### 5. 生成评估报告

按照 `agent-benchmark` skill 的报告模板生成结构化报告。

### 6. 发送报告

使用 `send_user_feedback` 将报告发送到配置的 chatId。

## 错误处理

1. 如果日志文件读取失败，跳过该文件继续处理
2. 如果 `send_user_feedback` 失败，记录日志并重试一次
3. 如果没有足够的交互数据，发送简要报告说明"数据不足"

## 验收标准

- [x] 能通过定时任务触发 (scheduler)
- [x] 能从聊天记录中提取 Agent 交互 (LLM 分析)
- [x] 能从 5 个维度评估 Agent 表现
- [x] 能对比不同模型/Framework (当数据可区分时)
- [x] 能描述独特的、无法量化的特性
- [x] 零代码侵入，不修改核心框架
