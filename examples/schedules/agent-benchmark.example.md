# Agent Framework 性能赛马周报

每周一 09:00 自动分析各 Agent 的服务质量，生成对比报告。

**关联 Issue**: #1334

## 配置说明

**重要**: 使用前请将 `chatId` 替换为实际的飞书 Chat ID，并将 `enabled` 设为 `true`。

## 执行步骤

### 1. 获取所有聊天记录文件

```bash
ls workspace/chat/*.md 2>/dev/null || ls workspace/logs/**/*.md 2>/dev/null || echo "No chat files found"
```

如果 `workspace/chat/` 和 `workspace/logs/` 目录均不存在或为空，跳过本次执行。

### 2. 读取聊天记录

使用 `Glob` 工具查找所有日志文件：
```
Glob workspace/chat/*.md
Glob workspace/logs/**/*.md
```

对于最近 7 天的日志文件，使用 `Read` 工具读取内容。

### 3. 分析 Agent 性能

对于每个聊天的交互记录，提取以下评估维度：

#### 3.1 响应效率
- 从消息时间戳计算首次响应时间
- 统计完成任务所需的对话轮次
- 评估响应的简洁性和针对性

#### 3.2 任务完成度
- 分析对话最终结果（用户是否得到满意的回答）
- 统计成功完成 / 部分完成 / 未完成的任务比例
- 记录典型的完成和失败案例

#### 3.3 用户满意度
- 识别正面反馈信号：感谢、好评、确认
- 识别负面反馈信号：纠正、不满、放弃
- 计算满意度比率

#### 3.4 工具使用效率
- 统计工具调用次数
- 评估工具调用的必要性和准确性
- 统计工具调用错误率

#### 3.5 错误模式
- 识别常见的错误类型和频率
- 分析错误恢复策略的效果
- 统计用户被迫手动纠正的次数

#### 3.6 独特特性（无法赛马的部分）
- 每个Agent独有的优势和能力
- 特定场景下的出色表现
- 无法量化的质量差异

### 4. 生成赛马报告

按照以下格式生成报告：

```markdown
## 🏁 Agent Framework 性能赛马周报

**分析时间**: [当前时间]
**分析范围**: 最近 7 天
**分析会话数**: [Number]
**涉及聊天**: [Number of chats]

---

### 📊 总体评分

| 维度 | [Agent A] | [Agent B] | [Agent C] |
|------|-----------|-----------|-----------|
| 响应效率 | ⭐⭐⭐⭐ | ⭐⭐⭐ | — |
| 任务完成度 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | — |
| 用户满意度 | ⭐⭐⭐⭐ | ⭐⭐⭐ | — |
| 工具使用效率 | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | — |
| 错误率 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | — |

[详细分析...]

[每个Agent的独特优势...]
```

### 5. 发送报告

使用 `send_user_feedback` 将报告发送到配置的 chatId：

```
send_user_feedback({
  content: [报告内容],
  format: "text",
  chatId: [配置的 chatId]
})
```

## 错误处理

1. 如果聊天记录文件读取失败，跳过该文件继续处理其他文件
2. 如果数据不足以进行有意义的比较（某个Agent少于3个会话），在报告中标注"数据不足"
3. 如果没有任何可分析的数据，不发送消息
4. 如果 `send_user_feedback` 失败，记录日志并重试一次

## 验收标准

- [x] 能定时触发分析 (通过 scheduler)
- [x] 能从聊天记录中提取多维度性能指标 (通过 LLM 分析)
- [x] 能对比不同 Agent 的表现 (结构化报告)
- [x] 能识别独特特性 (定性分析)
- [x] 零代码侵入 (纯外部 skill)
