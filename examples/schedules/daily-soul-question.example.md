# 每日灵魂拷问

每天定时分析聊天/工作记录，识别值得讨论的话题，发出开放式的灵魂拷问。

## 配置说明

**重要**:
1. 使用前请将 `chatId` 替换为话题群的 Chat ID
2. 将 `enabled` 设为 `true` 启用任务
3. 建议使用 `/create-topic` 命令创建话题群，或使用 `/mark-topic` 将现有群标记为话题群

## 核心场景

```
每日定时触发 (默认 09:00)
    ↓
分析聊天/工作记录
    ↓
识别值得讨论的话题：
- 有趣的决策
- 潜在的改进点
- 值得反思的问题
    ↓
发送灵魂拷问到话题群
    ↓
（不等待回复，开放式讨论）
```

## 执行步骤

### 1. 获取所有聊天记录

```bash
ls workspace/chat-logs 2>/dev/null || echo "No chat logs found"
```

如果 `workspace/chat-logs/` 目录不存在或为空，跳过本次执行。

### 2. 读取最近一天的聊天记录

对于每个聊天目录：

```bash
# 获取今天的日志文件
find workspace/chat-logs -name "$(date +%Y-%m-%d).md" -type f
```

读取文件内容，分析：
- 用户请求的任务类型
- Agent 完成的工作
- 遇到的问题和解决方案
- 做出的技术决策

### 3. 识别值得讨论的话题

从聊天记录中提取：

**优先级高的话题**:
- 🤔 **技术决策**: 为什么选择方案 A 而不是方案 B？
- 💡 **改进建议**: 有没有更好的实现方式？
- 🔍 **潜在问题**: 这个设计是否有隐患？
- 📚 **学习机会**: 这里涉及哪些值得深入的技术点？

**排除的话题**:
- 敏感信息（密码、密钥、个人数据）
- 已经明确解决的问题
- 过于简单的任务

### 4. 生成灵魂拷问

基于识别到的话题，生成 1-2 个开放式的灵魂拷问：

```markdown
🤔 今日灵魂拷问

分析今天的工作记录，发现一个值得思考的问题：

「{话题描述}」

**背景**：{上下文说明}

**思考点**：
- {思考问题 1}
- {思考问题 2}

欢迎在群里讨论 👇
```

### 5. 发送到话题群

使用 `send_user_feedback` 将灵魂拷问发送到配置的 chatId：

- 格式: text
- 内容: 生成的灵魂拷问
- chatId: 配置的话题群 ID

### 6. 记录历史

将发送的灵魂拷问追加到 `workspace/data/soul-question-history.json`：

```json
{
  "history": [
    {
      "date": "2026-03-06T09:00:00.000Z",
      "chatId": "oc_xxx",
      "topic": "技术决策讨论",
      "question": "为什么选择方案 A 而不是方案 B？"
    }
  ]
}
```

## 不发送的情况

跳过以下情况：
- 没有找到聊天记录
- 聊天记录中没有值得讨论的话题
- 今天已经发送过灵魂拷问（检查历史记录）

## 示例输出

```
🤔 今日灵魂拷问

分析今天的工作记录，发现一个有趣的问题：

「在处理用户认证模块时，我们选择了 JWT 而不是 Session。

这个决策是否正确？有没有更好的选择？」

**背景**：今天处理了 3 个与认证相关的任务，都使用了 JWT 方案。

**思考点**：
- JWT 的安全性如何保证？
- 是否需要支持 Token 撤销？
- 分布式场景下有什么注意事项？

欢迎在群里讨论 👇
```

## 与其他功能的区别

| 功能 | 交互类型 | 预期响应 | 典型场景 |
|------|----------|----------|----------|
| PR Review | 阻塞 | 必须回复 | 代码审查 |
| 每日回顾 | 非阻塞 | 触发任务 | 工作总结 |
| **灵魂拷问** | **开放式** | **不预期响应** | **话题讨论** |
