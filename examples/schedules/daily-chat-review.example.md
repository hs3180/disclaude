# 每日聊天回顾分析

每天 20:00 回顾所有聊天记录，识别重复问题模式，生成改进报告。

**关联 Issue**: #700
**Milestone**: 0.4.1 离线提问

## 配置说明

**重要**: 使用前请将 `chatId` 替换为实际的飞书 Chat ID，并将 `enabled` 设为 `true`。

## 执行步骤

### 1. 获取所有聊天日志文件

```bash
ls workspace/logs/**/*.md 2>/dev/null || echo "No log files found"
```

如果 `workspace/logs/` 目录不存在或为空，跳过本次执行。

### 2. 读取聊天日志

使用 `Glob` 工具查找所有日志文件：
```
Glob workspace/logs/**/*.md
```

对于最近 7 天的日志文件，使用 `Read` 工具读取内容。

### 3. 分析聊天记录

分析所有聊天记录，识别以下模式：

#### 3.1 重复用户请求
- 同一或类似的问题被多次询问
- 模式示例：用户多次问"检查 issues"、"看看 PR 状态"
- 筛选条件：至少出现 3 次

#### 3.2 人工纠正模式
- 用户反复纠正 Agent 的输出
- 模式示例："不对，应该是..."、"改成..."
- 筛选条件：至少出现 2 次

#### 3.3 错误模式
- 类似的错误消息频繁出现
- 模式示例："失败"、"错误"、"无法"、"error"
- 筛选条件：至少出现 3 次

#### 3.4 功能请求
- 用户表达对某功能的期望
- 模式示例："希望能..."、"如果有..."、"可以加..."
- 筛选条件：至少出现 2 次

### 4. 生成分析报告

按照以下格式生成报告：

```markdown
## 📊 每日聊天回顾分析报告

**分析时间**: [当前时间]
**分析范围**: 最近 7 天
**聊天数量**: [分析的聊天数]
**消息数量**: [总消息数]

---

### 🔴 重复问题 (需要关注)

#### 问题 1: [问题标题]
- **出现次数**: X 次
- **涉及聊天**: [聊天 ID 列表]
- **典型请求**:
  > [示例用户请求原文]

- **建议行动**:
  - [ ] 创建定时任务自动化
  - [ ] 创建 skill 简化操作
  - [ ] 创建 issue 记录需求

---

### 🟡 改进机会 (可选优化)

#### 机会 1: [改进标题]
- **描述**: [具体描述]
- **潜在收益**: [预期收益]

---

### ✅ 运行良好

- [运行良好的功能/流程列表]

---

### 📋 建议的下一步

1. **立即行动**: [高优先级项目]
2. **计划中**: [中优先级项目]
3. **观察**: [低优先级项目]
```

### 5. 发送报告

使用 `send_user_feedback` 将报告发送到配置的 chatId：

```
send_user_feedback({
  content: [报告内容],
  format: "text",
  chatId: [配置的 chatId]
})
```

### 6. (Phase 2) 发起离线讨论

**注意**: 此功能依赖 #631 的实现，目前为可选步骤。

对于高优先级问题，可以使用 `leave_note` 发起离线讨论：

```
leave_note({
  question: "关于检测到的重复问题「XXX」，您希望如何处理？",
  context: "过去 7 天中出现了 X 次...",
  chatId: [配置的 chatId],
  tags: ["daily-review", "improvement"]
})
```

## 错误处理

1. 如果日志文件读取失败，跳过该文件继续处理其他文件
2. 如果 `send_user_feedback` 失败，记录日志并重试一次
3. 如果没有检测到任何模式，发送简要报告说明"无异常模式"
