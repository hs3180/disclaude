# 每日聊天回顾分析

每天晚上 20:00 回顾聊天记录，识别需要改进的问题，输出分析报告。

## 配置说明

**重要**: 使用前请将 `chatId` 替换为实际的飞书 Chat ID，并将 `enabled` 设为 `true`。

## 背景与目标

这是 0.4.1 MVP 用例的核心功能，通过每日回顾聊天记录，主动发现需要改进的问题。

**核心场景**:
```
每日定时触发
    ↓
回顾所有聊天记录
    ↓
分析识别：
- 被人类反复要求的问题
- 多次人工纠正的问题
- 高频出现的错误
    ↓
选择最突出的问题
    ↓
输出分析报告
```

## 执行步骤

### 1. 获取聊天记录文件

```bash
ls -la workspace/logs/ 2>/dev/null | head -20
```

如果 `workspace/logs/` 目录不存在或为空，跳过本次执行。

### 2. 读取最近的聊天记录

获取最近 3 天的聊天记录：

```bash
find workspace/logs -name "*.md" -mtime -3 2>/dev/null | head -50
```

对于每个聊天记录文件，读取内容进行分析。

### 3. 分析识别问题模式

仔细分析聊天记录，识别以下类型的问题：

#### 3.1 被人类反复要求的问题

**特征**:
- 用户多次重复相同或相似的请求
- 用户表达不满或 frustration
- 用户说"再试一次"、"还是不对"等

**示例模式**:
- "帮我生成一个 PR" → 失败 → "再试一次" → 失败
- "检查代码" → 结果不满意 → "重新检查"

#### 3.2 多次人工纠正的问题

**特征**:
- Agent 输出后，用户需要多次修正
- 用户说"不对"、"应该是"、"改一下"
- Agent 的理解与用户意图不一致

**示例模式**:
- Agent: "创建了文件 X" → User: "应该创建文件 Y"
- Agent: "使用方案 A" → User: "用方案 B"

#### 3.3 高频出现的错误

**特征**:
- 相同类型的错误重复出现
- 工具调用失败
- 超时或资源限制

**示例模式**:
- "Bash command timeout"
- "File not found"
- "Permission denied"

### 4. 评估问题优先级

对识别到的问题按以下标准排序：

| 优先级 | 条件 |
|--------|------|
| 🔴 **高** | 出现 3 次以上 + 影响核心功能 |
| 🟡 **中** | 出现 2 次以上 + 用户明确表达不满 |
| 🟢 **低** | 出现 1-2 次 + 影响较小 |

使用 `send_user_feedback` 发送分析报告：

## 错误处理

1. 如果聊天记录目录不存在，跳过本次执行
2. 如果读取文件失败，跳过该文件继续处理
3. 如果 `send_user_feedback` 失败，记录日志并重试一次
4. 如果没有识别到问题，发送一条简短的无问题报告

## 无问题报告示例

```markdown
## 📊 每日聊天回顾分析报告

**分析范围**: 最近 3 天的聊天记录

继续保持良好的工作状态！
```
