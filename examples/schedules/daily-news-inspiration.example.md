# 每日新闻灵感提问 — 基于时事的自发交互

每天上午 9:00 使用 `daily-news-inspiration` skill 浏览最新新闻，提取灵感，生成自然提问并发送到目标群组。

## 配置说明

**重要**: 使用前请将 `chatId` 替换为实际的飞书 Chat ID（可以是话题群或普通群），并将 `enabled` 设为 `true`。

## 执行步骤

### 1. 执行前检查

检查今天是否已经发送过新闻提问：

```bash
cat workspace/chat/{chatId}.md 2>/dev/null | tail -100 || echo "No chat history"
```

- 如果今天已有新闻灵感提问 → 跳过（避免重复）
- 如果群内最近 2 小时内有超过 10 条消息 → 跳过（群组已活跃，不打扰）

### 2. 浏览最新新闻

使用 WebSearch 浏览今日社会和科技新闻：

```
WebSearch("今日社会新闻热点 2026")
WebSearch("今日科技新闻 AI 2026")
```

按星期偏好不同新闻类型：

| 星期 | 偏好类型 |
|------|----------|
| 周一 | 科技前沿 / AI 动态 |
| 周二 | 社会热点 / 政策趋势 |
| 周三 | 产品发布 / 行业变化 |
| 周四 | 开源动态 / 技术社区 |
| 周五 | 轻松话题 / 跨界创新 |

### 3. 选取新闻 + 生成提问

从搜索结果中选取**一条**新闻，以真实用户口吻生成一个自然的问题。

**选取标准**：
- 偏好科技/AI 相关（与用户画像匹配）
- 有讨论深度（不仅是"发生了某事"）
- 避免纯负面或争议性话题
- 避免需要专业知识的话题

**问题特征**：
- 自然口语化，不要正式或新闻腔
- 开放式（邀请讨论，非是/否）
- 包含 1-2 句背景信息，使问题自洽
- 类型多样（分析类、观点类、事实类、创意类、比较类）

### 4. 发送提问

使用 `mcp__channel-mcp__send_text` 发送到目标群组：

```
mcp__channel-mcp__send_text({
  text: "{generated question}",
  chatId: "{配置的chatId}"
})
```

### 5. 观察与评估（内部，不发送）

发送后，内部评估：
- disclaude 是否理解了新闻背景？
- 回复是否相关且有帮助？
- 有无值得报告的问题？

如发现显著问题，创建 GitHub issue 反馈。

## 质量要求

### 好的提问:
- ✅ 基于真实、当天的新闻
- ✅ 像真人看到新闻后想讨论的样子
- ✅ 开放式，有讨论空间
- ✅ 自洽（包含足够背景）
- ✅ 类型多样

### 避免:
- ❌ 是/否问题
- ❌ 需要专业知识才能回答
- ❌ 纯负面/悲剧性事件
- ❌ 模板化或重复的问题
- ❌ 一天多发多条
- ❌ 政治敏感或争议性话题

## 示例输出

```markdown
刚看到 GPT-5 发布了，据说支持多模态推理，能同时理解图片和文字的复杂关系了。你觉得这种能力对日常开发工作会有实际影响吗？还是说更多是概念上的进步？
```

## 错误处理

1. 如果 WebSearch 无法获取结果 → 记录日志，跳过本次执行
2. 如果找不到合适的新闻 → 不发送消息
3. 如果 `send_text` 失败 → 记录日志，不重试（避免重复发送）
