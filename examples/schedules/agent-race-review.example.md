# Agent Framework 质量评估

每周一 10:00 分析过去一周的聊天记录，评估各 Agent 框架/模型的服务质量。

**关联 Issue**: #1334
**使用 Skill**: agent-race-review

## 配置说明

**重要**: 使用前请将 `chatId` 替换为实际的飞书 Chat ID，并将 `enabled` 设为 `true`。

## 执行步骤

### 1. 获取所有聊天日志文件

```bash
ls workspace/logs/**/*.md 2>/dev/null || echo "No log files found"
```

如果 `workspace/logs/` 目录不存在或为空，跳过本次执行。

### 2. 读取聊天日志

使用 `Glob` 工具查找所有日志文件：
```
Glob workspace/logs/**/*.md
```

对于最近 7 天的日志文件，使用 `Read` 工具读取内容。

### 3. 识别不同 Agent 类型的交互

从聊天记录中识别以下 Agent 类型：
- **Issue Solver**: 定时任务触发的自主任务执行
- **PR Scanner**: PR 审查和讨论管理
- **Skill Agent**: 用户主动调用的各种 Skill
- **Chat Agent**: 常规对话交互
- **Schedule Agent**: 定时任务管理

### 4. 评估各维度

对每个 Agent 类型的交互，从以下维度评估：

#### 4.1 响应效率
- 用户请求到首次响应的时间
- 任务从开始到完成的总时间
- 多轮对话的平均轮次

#### 4.2 任务完成率
- 成功完成的任务比例
- 被用户放弃的任务比例
- 需要多轮修正才能完成的任务比例

#### 4.3 用户满意度
- 正面反馈（感谢、好评）
- 负面反馈（纠正、抱怨、重复提问）
- 主动推荐或表扬

#### 4.4 工具使用效率
- 平均每次任务的工具调用次数
- 工具调用成功率
- 无效/冗余工具调用比例

#### 4.5 错误率
- 任务执行失败次数
- 超时次数
- 重试次数

### 6. 发送报告

使用 `send_user_feedback` 将报告发送到配置的 chatId：

```
send_user_feedback({
  content: [报告内容],
  format: "text",
  chatId: [配置的 chatId]
})
```

## 错误处理

1. 如果日志文件读取失败，跳过该文件继续处理其他文件
2. 如果 `send_user_feedback` 失败，记录日志并重试一次
3. 如果没有足够的聊天记录，发送简要报告说明"数据不足，无法生成对比报告"
4. 如果某种 Agent 类型的交互少于 3 次，在报告中标注"样本不足"
