# Research Note — RESEARCH.md 研究状态文件管理

## Context

你正在 Research 模式中进行研究工作。本 Skill 负责管理 `RESEARCH.md` 文件的全生命周期：初始化、更新和归档。

**触发条件**：
- 进入 Research 模式时自动初始化 RESEARCH.md
- 研究过程中发现新信息时更新 RESEARCH.md
- 研究结束时归档结论

## Single Responsibility

- ✅ 创建和维护 RESEARCH.md 研究状态文件
- ✅ 跟踪研究目标、发现、待调查问题和结论
- ✅ 研究结束时归档研究文件
- ❌ 不执行实际研究（由 agentic-research skill 负责）
- ❌ 不管理 Research 模式切换（由 mode framework 负责）

## RESEARCH.md 文件规范

### 文件位置

- **路径**: `{research_workspace}/RESEARCH.md`
- **生命周期**: 随研究目录创建而初始化，随研究完成而归档

### 内容结构模板

```markdown
# {研究主题}

> 简要描述研究的目标和背景

## 研究目标

- [ ] 目标 1
- [ ] 目标 2

## 已收集的信息

### 发现 1
- **来源**: ...
- **关键内容**: ...
- **日期**: YYYY-MM-DD

### 发现 2
- **来源**: ...
- **关键内容**: ...
- **日期**: YYYY-MM-DD

## 待调查的问题

- [ ] 问题 1
- [ ] 问题 2

## 研究结论

（研究完成后填写）

## 相关资源

- [资源名称](链接)
```

## Workflow

### Phase 1: 初始化

**触发时机**: 研究开始时，检测到研究工作目录中不存在 RESEARCH.md

**步骤**:
1. 确认研究工作目录存在（即当前工作目录）
2. 检查是否已存在 `RESEARCH.md`
3. 如不存在，使用下方模板创建初始文件
4. 根据用户输入的研究主题和目标填充模板

**初始化模板**（使用 Write 工具创建）:

```
# {研究主题}

> {简要描述研究的目标和背景}
>
> 创建时间: {YYYY-MM-DD}
> 状态: 🟡 进行中

## 研究目标

- [ ] {从用户输入提取目标 1}
- [ ] {从用户输入提取目标 2}

## 已收集的信息

_（暂无发现，开始研究后将自动记录）_

## 待调查的问题

- [ ] {从用户输入提取待调查问题 1}
- [ ] {从用户输入提取待调查问题 2}

## 研究结论

_（研究完成后填写）_

## 相关资源

_（研究过程中收集的相关资源）_
```

**状态标记规范**:
| 状态 | 标记 | 含义 |
|------|------|------|
| 进行中 | 🟡 | 研究正在进行 |
| 已暂停 | 🟠 | 研究暂时中断 |
| 已完成 | 🟢 | 研究已完成并归档 |

### Phase 2: 自动更新

**触发时机**: 每次研究交互后（发现新信息、解决问题、获取资源）

**更新规则**:

| 事件 | 更新章节 | 操作 |
|------|----------|------|
| 发现新信息 | `## 已收集的信息` | 追加新发现条目 |
| 产生新问题 | `## 待调查的问题` | 追加新问题 |
| 问题已解决 | `## 已收集的信息` + `## 待调查的问题` | 移动条目（从未解决→已收集） |
| 目标已完成 | `## 研究目标` | 勾选对应 checkbox |
| 找到新资源 | `## 相关资源` | 追加资源链接 |
| 研究暂停 | 文件头部状态 | 更新状态为 🟠 |

**更新操作步骤**:

1. **读取当前 RESEARCH.md**（Read 工具）
2. **确定更新类型**（发现/问题/目标/资源）
3. **应用更新**（使用 Edit 工具精确修改对应章节）
4. **验证格式一致性**（确保 markdown 结构完整）

**发现条目格式**:
```markdown
### {发现标题}
- **来源**: {来源名称/URL}
- **关键内容**: {关键信息摘要}
- **日期**: YYYY-MM-DD
```

**更新注意事项**:
- ⚠️ 每次更新只修改相关章节，不要重写整个文件
- ⚠️ 保持时间倒序（最新发现排在最前面）
- ⚠️ 信息来源必须明确标注
- ⚠️ 区分事实和推论，推论用 *斜体* 标记

### Phase 3: 结论归档

**触发时机**: 用户明确表示研究完成，或所有研究目标已达成

**归档步骤**:

1. **填写研究结论**:
```markdown
## 研究结论

### 核心发现
{总结 3-5 条核心发现}

### 建议
{基于研究发现给出的建议}

### 未解决问题
{列出研究过程中未能完全解决的问题}

### 后续方向
{建议的后续研究方向}
```

2. **更新文件状态**: 将头部状态从 🟡 更新为 🟢

3. **清理待调查问题**:
   - 将所有未解决的问题移至"未解决问题"章节
   - 清空"待调查的问题"章节

4. **最终检查**:
   - [ ] 所有研究目标已标记（完成或未完成）
   - [ ] 研究结论已填写
   - [ ] 文件状态已更新为 🟢
   - [ ] 格式完整、无空白占位符

## 检测与触发规则

### 自动检测 RESEARCH.md 状态

进入研究工作目录时，检查 RESEARCH.md 的存在和状态：

```
检查流程:
1. 当前目录是否存在 RESEARCH.md？
   ├── 不存在 → 执行 Phase 1（初始化）
   └── 存在 → 读取状态标记
       ├── 🟡 进行中 → 准备 Phase 2（更新）
       ├── 🟠 已暂停 → 提示用户是否继续
       └── 🟢 已完成 → 提示用户是否开始新研究
```

### 与 Research Mode 的集成

当 Research Mode（Issue #1709）可用时，本 Skill 作为研究专用 skill 子集的一部分自动加载。Research Mode 的 `setupResearchWorkspace()` 会创建研究目录结构，本 Skill 负责在目录中创建和管理 RESEARCH.md。

## 输出文件

### RESEARCH.md (Required)

研究状态文件，位于研究工作目录根路径。

## DO NOT

- ❌ 重写整个 RESEARCH.md 文件（使用 Edit 工具精确更新）
- ❌ 在非研究工作目录中创建 RESEARCH.md
- ❌ 删除已有的研究发现（只追加，不删除）
- ❌ 在结论归档前将状态标记为已完成
- ❌ 忽略信息来源标注
