# 每日新闻灵感提问 — 定时执行

浏览最新新闻，提取灵感，生成一个自然的提问发送到当前群聊。

## 你的任务

### 第一步：执行前检查

检查今天是否已经发送过新闻提问：

```bash
cat workspace/chat/{targetChatId}.md 2>/dev/null | tail -100 || echo "No chat history"
```

- 如果今天已有新闻灵感提问 → 跳过（避免重复）
- 如果群内最近 2 小时内有超过 10 条消息 → 跳过（群组已活跃，不打扰）

### 第二步：浏览最新新闻

使用 WebSearch 浏览今天最新的社会新闻和科技新闻：

1. **社会新闻** (社会新闻): 重大事件、政策变化、文化趋势
2. **科技新闻** (科技新闻): 产品发布、AI 突破、行业趋势

搜索示例：
```
WebSearch("今日社会新闻热点 2026")
WebSearch("今日科技新闻 AI 2026")
```

### 主题轮换

按星期偏好不同新闻类型：

| 星期 | 偏好类型 |
|------|----------|
| 周一 | 科技前沿 / AI 动态 |
| 周二 | 社会热点 / 政策趋势 |
| 周三 | 产品发布 / 行业变化 |
| 周四 | 开源动态 / 技术社区 |
| 周五 | 轻松话题 / 跨界创新 |

### 第三步：选择一条新闻

从搜索结果中挑选 **一条** 新闻，要求：
- 真正有趣或引人思考
- 有深入讨论的潜力
- 与用户相关（技术敏感、使用 AI 工具）
- 非纯负面或标题党

选择偏好：
- 优先科技/AI 相关新闻（相关性更高）
- 优先有深度的话题（非纯事件）
- 避免纯政治或争议话题
- 避免需要专业领域知识的新闻

### 第四步：生成自然提问

以 **真实用户口吻** 写一个问题，像刚看到新闻想讨论一样：

- 自然、口语化的语气
- 展现真正的好奇心
- 开放式（引导讨论，非是非题）
- 包含 1-2 句背景上下文，使问题自足

提问格式：
```markdown
{1-2 句新闻背景，口语化}

{自然的追问或好奇}
```

问题类型轮换：

| 类型 | 描述 | 示例 |
|------|------|------|
| 分析类 | 深度分析趋势 | "X 的真正影响是什么？" |
| 观点类 | 对发展的看法 | "你怎么看 X 的做法？" |
| 事实类 | 具体细节 | "X 到底是怎么工作的？" |
| 创意类 | 创意推测 | "如果 X 应用到 Y 会怎样？" |
| 比较类 | 比较方案 | "X vs Y 哪个更合理？" |

### 第五步：发送提问

使用 `mcp__channel-mcp__send_text` 工具发送：

```
mcp__channel-mcp__send_text({
  text: "{生成的提问}",
  chatId: "{targetChatId}"
})
```

## 质量指南

### 好的问题:
- 基于真实、时事新闻
- 像真人提问
- 开放式，有讨论空间
- 自足（包含足够上下文）
- 类型和话题多样

### 避免:
- 是非题
- 需要专业知识才能回答
- 纯负面/悲剧事件
- 模板化或重复的问题
- 像测试而非真正好奇
- 一天发送多个问题（质量优先）
- 政治敏感或争议话题
- 标题党风格
