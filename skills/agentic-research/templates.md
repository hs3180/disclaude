# Research Report Templates

Pre-built templates for different research types. Use the appropriate template based on the research context.

---

## Template 1: Technical Investigation

Use for: Technology evaluation, architecture research, tool comparison, API analysis.

```markdown
# Technical Investigation: {topic}

> **Date**: {date}
> **Status**: Complete
> **Scope**: {brief scope description}

## Executive Summary

{2-3 sentences: what was investigated, key finding, recommendation}

## Background

{Why this investigation was initiated, what problem it addresses}

## Findings

### {Technology/Tool/Approach A}

**Overview**: {What it is, how it works}

**Strengths**:
- ✅ {strength 1}
- ✅ {strength 2}

**Weaknesses**:
- ❌ {weakness 1}
- ❌ {weakness 2}

**Evidence**:
- {specific data points, benchmarks, code examples}

### {Technology/Tool/Approach B}

{Same structure as above}

## Comparison

| Criteria | Option A | Option B | Option C |
|----------|----------|----------|----------|
| Performance | ... | ... | ... |
| Ease of use | ... | ... | ... |
| Community | ... | ... | ... |
| Cost | ... | ... | ... |

## Recommendation

{Clear recommendation with justification}

## Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| {risk} | High/Med/Low | High/Med/Low | {strategy} |

## Next Steps

1. {Action item 1}
2. {Action item 2}

## Sources

1. [{Source}]({url})
```

## Template 2: Analysis Report

Use for: Market analysis, competitive analysis, trend analysis, data-driven insights.

```markdown
# Analysis Report: {topic}

> **Date**: {date}
> **Status**: Complete
> **Methodology**: {data collection method}

## Executive Summary

{Key findings in 2-3 sentences}

## Context

{Background and motivation for this analysis}

## Data Overview

{Description of data sources, time range, coverage}

## Key Findings

### Finding 1: {Finding Title}

**Observation**: {What the data shows}

**Significance**: {Why this matters}

**Supporting Data**:
- {data point 1}
- {data point 2}

### Finding 2: {Finding Title}

{Same structure}

## Trends & Patterns

{Identify trends, correlations, anomalies}

## Implications

### For {Stakeholder Group A}
- {Implication 1}
- {Implication 2}

### For {Stakeholder Group B}
- {Implication 1}

## Recommendations

1. **{Recommendation}**: {Rationale}
2. **{Recommendation}**: {Rationale}

## Limitations

- {Data limitations}
- {Methodology constraints}
- {Time/bias considerations}

## Sources

1. [{Source}]({url}) — {accessed date, reliability note}
```

## Template 3: Literature Review

Use for: Academic research, paper survey, knowledge synthesis, state-of-the-art review.

```markdown
# Literature Review: {topic}

> **Date**: {date}
> **Status**: Complete
> **Papers Reviewed**: {N}
> **Date Range**: {start} — {end}

## Abstract

{250-word summary of the review}

## Introduction

{Research context, why this topic matters, review scope}

## Methodology

{Search strategy, inclusion/exclusion criteria, databases used}

## Thematic Analysis

### Theme 1: {Theme Title}

**Overview**: {What this theme covers}

**Key Papers**:

| Paper | Authors | Year | Key Contribution |
|-------|---------|------|-----------------|
| [{Title}]({url}) | {Authors} | {Year} | {Contribution} |
| [{Title}]({url}) | {Authors} | {Year} | {Contribution} |

**Synthesis**: {How these papers relate, agreements and debates}

### Theme 2: {Theme Title}

{Same structure}

## Research Gaps

- {Identified gap 1}
- {Identified gap 2}

## Emerging Directions

{New trends, promising approaches, open questions}

## Conclusion

{Summary of the field's current state}

## References

1. {Authors} ({Year}). {Title}. {Journal/Conference}. [{URL}]({url})
2. ...
```

## Template 4: Summary Report

Use for: Quick fact-finding, sprint research, time-boxed investigations.

```markdown
# Quick Research: {topic}

> **Date**: {date}
> **Time Spent**: ~{N} minutes
> **Confidence**: High/Medium/Low

## TL;DR

{1-2 sentence answer to the original question}

## What I Found

### Answer

{Direct answer to the research question}

### Key Points

1. **{Point 1}**: {brief explanation}
2. **{Point 2}**: {brief explanation}
3. **{Point 3}**: {brief explanation}

### Caveats

- {Things to be aware of}
- {Uncertainties}

## If You Need More

- {Suggested deeper research topics}
- {Additional sources to consult}

## Sources

1. [{Source}]({url})
```

---

## Template Selection Guide

| If research is... | Use template... | Expected length |
|-------------------|-----------------|-----------------|
| Evaluating technologies | Technical Investigation | Long (detailed) |
| Analyzing markets/trends | Analysis Report | Long (data-heavy) |
| Surveying papers/knowledge | Literature Review | Long (comprehensive) |
| Quick question answering | Summary Report | Short (concise) |
| Default / unsure | Technical Investigation | Medium |
