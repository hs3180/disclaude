# Deep Search 使用指南

## 快速开始

### Feishu/Lark 模式

直接发送消息：
```
deep search 2024年人工智能在医疗领域的应用
```

或使用更自然的表达：
```
帮我深入研究一下量子计算在金融领域的应用
```

### CLI 模式

```bash
# 使用 deep search skill
disclaude --prompt "deep search renewable energy storage technologies 2024"

# 自然语言触发
disclaude --prompt "Comprehensive research on CRISPR gene editing clinical trials"
```

## 执行流程

### 第 1 阶段：研究规划 (1-2 分钟)

AI 将：
1. 分析你的研究问题
2. 分解为 4-6 个研究维度
3. 为每个维度生成 3-5 组精准关键词
4. **自动继续执行搜索，无需等待确认**

### 第 2 阶段：搜索与发现 (3-5 分钟)

AI 将：
1. 使用 Google 搜索执行每组关键词
2. 识别权威垂直领域站点
3. 评估站点质量和相关性
4. 筛选 5-10 个高价值站点
5. **自动继续深度搜集，无需等待确认**

### 第 3 阶段：深度搜集 (10-20 分钟)

AI 将：
1. 逐个访问选定的垂直站点
2. 探索站点结构和内容
3. 提取关键信息、数据、统计
4. 跟踪相关链接和引用
5. 在域内交叉验证信息
6. **定期更新进度**

**你需要：** 等待深度搜集完成（AI 会定期报告进度）

### 第 4 阶段：综合报告 (2-3 分钟)

AI 将生成完整报告，包含：
- 执行摘要
- 各维度关键发现
- 跨域分析（共识、分歧、趋势）
- 数据与统计表格
- 推荐资源
- 完整来源列表
- 研究局限性和进一步建议

**你需要：** 阅读报告，提出问题或要求进一步研究

## 最佳实践

### ✅ 推荐的研究主题

**好的深度研究主题：**
- "2024年全球电动汽车市场趋势分析"
- "人工智能在药物发现中的最新进展"
- "Web3 技术在供应链管理中的应用"
- "量子密码学 vs 传统加密：安全性与可行性对比"
- "远程工作对员工生产力的长期影响研究"

**特征：**
- 具体的领域或行业
- 明确的时间范围（如 2024）
- 可深入分析的技术或商业问题
- 多维度可探索的话题

### ❌ 不适合的主题

**简单查询（用普通搜索）：**
- "什么是机器学习"（定义类问题）
- "Python 怎么安装"（操作指南）
- "今天的天气"（实时信息）

**需要本机操作的任务：**
- "读取我的代码文件"（需要 Read 工具）
- "运行测试"（需要 Bash 工具）
- "编辑配置"（需要 Edit 工具）

## 高级技巧

### 1. 优化研究问题

**太宽泛：**
```
"deep search AI"
```

**更好的表达：**
```
"deep search 大语言模型在客户服务中的应用案例与ROI分析 2024"
```

### 2. 指定研究维度

可以在请求中明确指出你关心的维度：
```
"deep search 固态电池技术，重点关注：技术原理、主要厂商、量产时间表、成本对比"
```

### 3. 限定时间范围

```
"deep search 远程工作工具，重点关注 2023-2024 年的新产品"
```

### 4. 地理范围限定

```
"deep search 欧洲的 AI 监管政策，重点关注 EU AI Act"
```

### 5. 结合垂直领域

```
"deep search CRISPR 临床试验，优先关注 FDA 批准的项目"
```

## 预期时间

|-----------|---------|-----------|---------|
| 简单主题 | 15-20 分钟 | 5-7 | ~1500 字 |
| 中等主题 | 25-35 分钟 | 7-10 | ~2500 字 |
| 复杂主题 | 40-60 分钟 | 10-15 | ~4000 字 |

```

```

### 扩展研究范围
```
"补充研究一下亚太市场的情况"
```

### 对比分析
```

```


### 验证信息
```
"报告中的这个结论可靠吗？帮我找更多来源验证"
```

## 故障排查

### 问题：AI 无法访问某些站点

**原因：**
- 站点需要登录
- 地理限制
- 反爬虫机制
- 网络连接问题

**解决方案：**
AI 会自动跳过无法访问的站点，并尝试其他来源

### 问题：报告信息不够深入

**原因：**
- 主题过于新兴，资料有限
- 垂直站点质量不高
- 关键词不够精准

**解决方案：**
提供更具体的关键词或指定已知的权威站点

### 问题：研究时间过长

**原因：**
- 主题范围太广
- 垂直站点内容过多

**解决方案：**
缩小研究范围，指定重点维度

## 与普通搜索的区别

| 维度 | 普通搜索 | Deep Search |
|------|---------|-------------|
| **工具** | WebSearch (单一) | Google + Playwright (组合) |
| **流程** | 一次性搜索 | 4 阶段系统流程 |
| **深度** | 表面信息 | 深入垂直站点挖掘 |
| **验证** | 无 | 多源交叉验证 |
| **报告** | 简单摘要 | 结构化综合报告 |
| **时间** | 秒级 | 15-60 分钟 |
| **适用** | 快速查询 | 深度研究 |
