# Research Task Improvements Design Document

> Issue: #1021 - docs: research 类任务的常见抱怨与改进方向
> Milestone: 0.5.0 Agentic Research
> Status: Draft
> Created: 2026-03-07

## 1. Overview

在 0.5.0 Agentic Research 开发过程中，收集了 research 类任务的常见抱怨。本文档记录这些问题及其改进方向，作为系统设计的参考。

### 1.1 Scope

本文档涵盖：
- Research 类任务的 6 大类常见问题
- 每类问题的具体表现
- 短期和长期解决方案

## 2. Problem Analysis

### 2.1 数据源问题 (Data Source Issues) :alert:

> 使用错误的不可信的数据源来做研究，提供专业指导后无法长期记住，甚至会在某个消息开始偷偷把数据换回 AI 更喜欢的

#### 表现

- 使用不可靠的数据源
- 接受指导后短期内正确，随后又恢复错误行为
- 倾向于选择"方便"而非"正确"的数据源

#### 解决方案

| 方案 | 描述 | 复杂度 |
|------|------|--------|
| **短期**: 白名单机制 | 配置可信数据源列表，Agent 优先使用 | 低 |
| **长期**: 智能数据源评估 | 基于历史反馈自动评估数据源可信度 | 高 |

---

### 2.2 数据处理问题 (Data Processing Issues) :alert:

> 不会自动清洗数据，不知道选择合理的数据规格，偷偷用模拟数据代替真实数据，不对原始数据预处理导致性能极差

#### 表现

- 跳过数据清洗步骤
- 选择不合适的数据格式/精度
- 用模拟数据代替真实数据（因为"更方便"）
- 原始数据未预处理导致分析失败或性能极差

#### 解决方案

| 方案 | 描述 | 复杂度 |
|------|------|--------|
| **短期**: 最佳实践清单 | 在 research skill 中包含数据处理检查清单 | 低 |
| **长期**: 自动化数据处理流水线 | 自动检测数据类型并应用合适的预处理 | 高 |

---

### 2.3 研究方向问题 (Research Direction Issues) :alert:

> 研究目标不清晰，在没有参考价值的问题上进行大量分析，忽略显而易见的直观分析，不理解可视化数据上的 insight，听不懂修改意见左右横跳

#### 表现

- 在无关紧要的细节上花费大量时间
- 忽略显而易见的结论
- 无法从可视化中获取 insight
- 修改意见执行不稳定，左右摇摆

#### 解决方案

| 方案 | 描述 | 复杂度 |
|------|------|--------|
| **短期**: 目标检查点 | 在关键步骤确认研究目标是否对齐 | 低 |
| **长期**: 研究价值评估模型 | 自动评估分析方向的价值 | 高 |

---

### 2.4 学习能力问题 (Learning Ability Issues) :alert:

> 不会主动学习相关研究，甚至遗忘自己前面的知识。很难自主举证。不会学习最佳实践

#### 表现

- 不主动查阅相关文献/资料
- 遗忘之前获得的知识
- 无法自主提供支撑证据
- 重复犯同样的错误

#### 解决方案

| 方案 | 描述 | 复杂度 |
|------|------|--------|
| **短期**: 知识库 | 持久化存储关键发现和结论 | 中 |
| **长期**: 持续学习系统 | 自动从反馈中学习并改进 | 高 |

---

### 2.5 知识混淆问题 (Knowledge Confusion Issues) :alert:

> 混淆类似知识，简单指正难以纠正，必须写成类 skill 知识才能学会一种方法

#### 表现

- 混淆相似但不同的概念
- 口头指正效果差
- 只有结构化的 skill 才能有效学习

#### 解决方案

| 方案 | 描述 | 复杂度 |
|------|------|--------|
| **短期**: Skill 模板 | 为常见混淆知识创建对比 skill | 中 |
| **长期**: 知识图谱 | 构建知识关系图谱，自动识别相似概念差异 | 高 |

---

### 2.6 Skill 选择过载问题 (Skill Selection Overload) :alert:

> skill 累积过多之后，agent 就无法总是做最优选择，因此而变成跟一个业务不熟的小二在一个巨大的菜单上点菜，因此不可能构建一个庞大 skill 库来解决任意专业问题

#### 表现

- skill 数量过多时，agent 难以准确选择最合适的 skill
- 类似不熟悉业务的侍者在巨大菜单上随机点菜
- 无法构建庞大 skill 库来解决任意专业问题

#### 核心矛盾

- skill 太少 → 专业能力不足
- skill 太多 → 选择困难，无法精准匹配

#### 解决方案

| 方案 | 描述 | 复杂度 |
|------|------|--------|
| **短期**: 动态加载 + 分层组织 | 按需加载相关 skill，分层/分类组织 | 中 |
| **长期**: 智能匹配 + 非 skill 知识库 | 智能匹配算法 + 考虑非 skill 化的知识存储方式 | 高 |

---

## 4. Implementation Priority

基于问题严重程度和实现复杂度，建议实现优先级：

### P0 - 必须解决 (阻塞用户任务)

1. **数据源白名单** - 防止使用不可信数据源
2. **数据处理最佳实践** - 提升分析质量

### P1 - 应该解决 (显著影响体验)

3. **目标检查点** - 避免研究方向偏移
4. **知识持久化** - 防止知识遗忘

### P2 - 可以优化 (长期改进)

5. **Skill 分层组织** - 解决选择过载
6. **知识对比模板** - 减少混淆
