# Agent 性能评估周报

每周一 9:00 分析过去一周的聊天记录，评估 Agent 服务质量，生成对比报告。

## 执行步骤

### 1. 调用 agent-race-report skill

使用 `agent-race-report` skill 执行分析：

分析过去 7 天的所有聊天记录，按照以下 5 个维度评估 Agent 表现：
1. **响应效率** — 响应速度和交互轮次
2. **任务完成度** — 是否一次完成任务
3. **用户满意度** — 用户反馈和纠正次数
4. **工具使用效率** — 工具调用是否合理
5. **错误模式** — 失败、重试、超时等

### 2. 分析范围

- 读取 `workspace/logs/` 目录下最近 7 天的所有聊天记录
- 识别每次完整的用户-Agent 交互
- 按维度打分并汇总

### 3. 生成报告

生成结构化的性能评估报告，包含：
- 总体概览评分表
- 最佳/最差交互案例
- 趋势对比（与上周数据）
- 具体改进建议

### 4. 发送报告

使用 `send_user_feedback` 将报告发送到配置的 chatId。

### 5. 记录历史

将分析结果追加到 `workspace/data/race-report-history.json`，用于后续趋势分析。

## 错误处理

1. 如果聊天记录为空，发送 "本周无聊天记录，跳过评估" 消息
2. 如果 `send_user_feedback` 失败，记录日志并重试一次
3. 如果历史文件不存在，创建新文件

## 配置说明

使用前需要修改：
1. `chatId`: 替换为实际接收报告的群组 ID
2. `enabled`: 设置为 `true` 启用
3. `cron`: 根据需要调整频率（默认每周一）
