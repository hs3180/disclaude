# Task Progress Monitor

定期检查所有活跃任务的进度，向相关群聊发送进度更新卡片。

## 配置

- **任务目录**: `workspace/tasks/`
- **执行间隔**: 每 5 分钟
- **阻塞模式**: 非阻塞（不阻止其他定时任务）

## 职责边界

- ✅ 扫描所有活跃任务目录
- ✅ 读取任务文件（task.md, evaluation.md, execution.md）
- ✅ 计算任务状态和进度
- ✅ 向任务所属群聊发送进度卡片
- ✅ 识别卡住的任务（>30 分钟无更新）
- ❌ 不修改任何任务文件（只读）
- ❌ 不创建或删除任务
- ❌ 不执行任务

## 执行步骤

### Step 1: 扫描任务目录

使用 Glob 工具扫描 `workspace/tasks/*/task.md`，获取所有任务。

### Step 2: 筛选活跃任务

对每个任务检查状态文件：
- 有 `final_result.md` → 已完成，跳过
- 有 `failed.md` → 已失败，跳过
- 创建时间 < 1 分钟 → 太新，跳过
- 其余 → 活跃任务，继续分析

### Step 3: 读取任务信息

对每个活跃任务：
1. 读取 `task.md` 提取标题、Chat ID、创建时间
2. 读取最新 `iterations/iter-N/evaluation.md` 获取评估状态
3. 读取最新 `iterations/iter-N/execution.md` 获取执行进展
4. 计算已用时间和迭代次数

### Step 4: 发送进度更新

对有有意义更新的任务，向其 Chat ID 发送进度卡片。

**有意义更新**的判断：
- 有新的迭代（相比上次检查）
- 任务可能卡住（>30 分钟无更新）
- 任务刚完成或刚失败

## 状态分类

| 条件 | 状态 | 图标 |
|------|------|------|
| 有 iterations 目录 | 执行中 | 🔄 |
| 无 iterations 目录 | 等待中 | ⏳ |
| >30 分钟无更新 | 可能卡住 | ⚠️ |

## 错误处理

| 场景 | 处理方式 |
|------|----------|
| 任务文件损坏 | 跳过该任务 |
| Chat ID 缺失 | 跳过发送，记录日志 |
| 无活跃任务 | 不发送任何消息 |
| Glob 返回空 | 不发送任何消息 |

## 注意事项

1. **只读访问**: 不修改任何任务文件
2. **避免噪音**: 只在有有意义更新时发送
3. **独立运行**: 不阻塞主任务流程
4. **默认禁用**: 需要手动启用此 schedule
