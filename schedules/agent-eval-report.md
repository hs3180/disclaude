# Agent 性能评估周报

每周一上午 9:00 自动分析所有聊天记录，生成 Agent/Framework 性能评估报告。

## 执行步骤

### 1. 收集聊天记录

使用 Glob 查找所有聊天日志文件：

```
Glob: workspace/logs/**/*.md
```

过滤最近 7 天的日志文件（按文件名中的日期判断）。

如果没有任何日志文件，跳过本次执行。

### 2. 执行分析

调用 `agent-eval` skill 进行分析：

1. 读取每个聊天日志文件
2. 按五个维度分析每个聊天：
   - **响应效率**：从消息时间戳计算响应时间
   - **任务完成度**：分析对话模式判断任务是否完成
   - **用户满意度**：识别正/负反馈信号
   - **工具使用效率**：统计工具调用模式
   - **错误率**：统计错误发生和恢复情况
3. 跨聊天对比，生成排名和趋势

### 3. 生成报告

生成包含以下内容的结构化报告：
- 总览指标表（含周环比趋势）
- 各维度详细分析
- 重点发现（亮点、需关注、改进建议）
- 历史趋势（如存在历史数据）

### 4. 发送报告

使用 `send_user_feedback` 将报告发送到配置的 chatId。

### 5. 保存评估数据

将本次评估的原始指标数据追加到 `workspace/data/agent-eval-history.json`，保留最近 12 周的数据用于趋势分析。

## 配置说明

使用前需要修改：
1. `chatId`: 替换为实际的目标群组 ID
2. `enabled`: 设置为 `true` 启用
3. `cron`: 根据需要调整执行时间

## 错误处理

1. 如果聊天日志读取失败，跳过该文件继续处理其他文件
2. 如果 `send_user_feedback` 失败，记录错误并重试一次
3. 如果数据量不足（少于 10 条消息），发送简要状态而非完整报告
4. 如果历史数据文件损坏，重新创建而非中断执行
