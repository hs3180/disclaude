# BBS 话题自动发起

工作日每天 9:00 和 15:00 自动分析群组活跃度，在需要时发起话题。

## 执行步骤

### 1. 检查最近活跃度

读取当前群组的聊天记录：

```bash
cat workspace/chat/{chatId}.md 2>/dev/null | tail -100 || echo "No chat history"
```

**活跃度判断标准**：
- 如果最近 4 小时内有超过 5 条消息，跳过本次（群组已活跃）
- 如果最近 4 小时内已发送过话题，跳过本次（避免重复）

### 2. 分析讨论主题

识别最近讨论的主题：
- 技术相关（代码、架构、工具）
- 项目相关（进度、决策、问题）
- 闲聊相关（生活、兴趣、新闻）

### 3. 生成话题

使用 `bbs-topic-initiator` skill 生成话题：

根据分析结果选择话题类型：
- **技术前沿**：如果最近技术讨论较多
- **经验分享**：如果遇到问题需要讨论
- **轻松闲聊**：如果群组长期冷清

### 4. 发送话题

使用 `send_user_feedback` 发送到当前 chatId：

```
send_user_feedback({
  chatId: "{chatId}",
  message: "{generated_topic}"
})
```

## 话题类型轮换

为避免重复，按以下顺序轮换话题类型：

| 星期 | 话题类型 |
|------|----------|
| 周一 | 技术前沿 / 本周计划 |
| 周二 | 经验分享 / 最佳实践 |
| 周三 | 问题讨论 / 技术挑战 |
| 周四 | 工具推荐 / 效率提升 |
| 周五 | 轻松闲聊 / 本周总结 |

## 错误处理

1. 如果聊天记录读取失败，生成一个通用话题
2. 如果 `send_user_feedback` 失败，记录日志
3. 如果群组已活跃或刚发过话题，静默跳过

## 配置说明

使用前需要修改：
1. `chatId`: 替换为实际的 BBS 群组 ID
2. `enabled`: 设置为 `true` 启用
3. `cron`: 根据需要调整时间
