# Agent Framework 服务质量评估

每周一 09:00 分析各个 chat 的历史聊天记录，评估并对比不同 Agent 框架/模型的服务质量。

**关联 Issue**: #1334

## 配置

- **日志目录**: `workspace/logs/`
- **执行间隔**: 每周一 09:00
- **分析范围**: 最近 7 天
- **历史数据**: `workspace/data/framework-review-history.json`

## 执行步骤

### 1. 获取所有聊天日志文件

```bash
ls workspace/logs/**/*.md 2>/dev/null || echo "No log files found"
```

如果 `workspace/logs/` 目录不存在或为空，跳过本次执行。

### 2. 读取聊天日志

使用 `Glob` 工具查找所有日志文件：
```
Glob workspace/logs/**/*.md
```

筛选最近 7 天的日志文件，使用 `Read` 工具读取内容。

### 3. 识别 Agent 框架信息

对每段对话，识别以下信息：
- **Provider**: `anthropic`, `openai`, `google` 等（从日志中的元数据或系统消息判断）
- **Model**: 具体模型名称
- **Agent 类型**: skill agent, task agent, chat agent 等
- **任务类型**: 编码、分析、写作、问答等

### 4. 多维度评估

对每个识别到的框架/模型，按以下维度评估：

| 维度 | 指标 | 数据来源 |
|------|------|---------|
| ⚡ 响应效率 | 消息间隔时间、首 token 延迟 | 时间戳 |
| ✅ 任务完成度 | 一次完成率、多轮完成率 | 对话轮次 + 结果 |
| 😊 用户满意度 | 正面/负面反馈比例 | 用户反馈关键词 |
| 🔧 工具使用效率 | 工具调用次数、相关性 | 工具调用记录 |
| ❌ 错误率 | 工具失败、任务失败、重试 | 错误日志 |

每个维度使用 ⭐ 评分（1-5 星）。

### 5. 识别独特特性

> 这是"赛马"无法量化的部分，也是最有价值的洞察。

对每个框架识别：
- 无法被数字量化的独特能力
- 特殊的交互风格
- 擅长的领域
- 值得注意的行为模式

### 6. 读取历史数据（如存在）

```bash
cat workspace/data/framework-review-history.json 2>/dev/null || echo "No history file"
```

如果存在历史数据，生成本期 vs 上期的趋势对比。

### 7. 生成报告

按照 `framework-review` skill 中定义的报告格式生成评估报告。

报告核心包含：
- **综合评分表**: 各框架各维度的星级评分
- **详细分析**: 每个框架的优势、不足、典型交互
- **独特特性**: 无法量化的定性分析
- **趋势变化**: 与上期对比
- **建议**: 任务分配优化、改进方向

### 8. 保存历史数据

将本次评估结果追加到 `workspace/data/framework-review-history.json`：

```json
{
  "history": [
    {
      "date": "2026-04-18T09:00:00.000Z",
      "period": "2026-04-11 to 2026-04-18",
      "chatCount": 0,
      "messageCount": 0,
      "frameworks": {},
      "highlights": []
    }
  ]
}
```

### 9. 发送报告

使用 `send_user_feedback` 将报告发送到配置的 chatId。

如果数据不足（少于 5 个可分析对话），发送简要报告说明数据不足，并建议延长评估周期。

## 错误处理

1. 如果日志文件读取失败，跳过该文件继续处理其他文件
2. 如果 `send_user_feedback` 失败，记录日志并重试一次
3. 如果历史数据文件损坏，创建新文件（不丢失本次分析结果）
4. 如果没有检测到可分析数据，发送简要状态报告

## 注意事项

1. **零代码侵入**: 本 schedule 仅分析聊天记录，不修改任何核心 Agent 代码
2. **数据驱动**: 所有评估基于实际聊天记录，不臆造数据
3. **定性 + 定量**: 既包含量化评分，也包含无法量化的独特特性分析
4. **历史趋势**: 通过持续执行积累数据，生成趋势分析
5. **不创建新 Schedule**: 遵守定时任务执行规则
