# Agent Framework 赛马 - 每周报告

每周一早上 9 点分析所有聊天的 Agent 执行质量，生成框架对比报告。

> Related: #1334

## 执行步骤

### 1. 发现数据源

#### 1.1 聊天记录

```bash
find workspace/chat -name "*.md" -mtime -7 2>/dev/null | head -50
```

如果没有找到文件，尝试旧版路径：

```bash
ls workspace/chat/*.md 2>/dev/null | head -50
```

如果仍然没有文件，退出本次执行（数据不足）。

#### 1.2 结构化日志（补充数据）

```bash
ls logs/disclaude-combined.log 2>/dev/null && echo "FOUND" || echo "NOT_FOUND"
```

如果存在，提取 provider 信息：

```bash
grep -o '"provider":"[^"]*"' logs/disclaude-combined.log 2>/dev/null | sort | uniq -c | sort -rn | head -10
```

#### 1.3 Agent 配置

```bash
cat disclaude.config.yaml 2>/dev/null | grep -A10 'agent:' || echo "No config found"
```

### 2. 分析聊天记录

对于每个发现的聊天记录文件：

1. **读取文件内容**（使用 Read 工具）
2. **跳过不满足条件的文件**：
   - 消息少于 10 条的文件
   - 纯系统消息的文件
   - 定时任务执行记录

3. **提取 5 个核心维度**：

   **a) 响应效率**
   - 计算用户消息 (`👤`) 到 Bot 回复 (`🤖`) 的时间差
   - 统计平均值、中位数、P95

   **b) 任务完成度**
   - 识别任务完成信号（"完成"、"成功"、"已提交"等）
   - 识别任务失败信号（"失败"、"错误"、"无法"等）
   - 计算完成率

   **c) 用户满意度**
   - 正面信号："谢谢"、"很好"、"完美"、"thanks"
   - 负面信号："不对"、"不满意"、"还是不行"、"重来"
   - 纠正模式：用户反复纠正 Bot 输出

   **d) 工具使用效率**
   - 统计工具调用次数（通过 Bot 消息中的模式识别）
   - 识别冗余调用（同一工具重复调用类似参数）

   **e) 错误率**
   - "Error"、"failed"、"exit code 1"、"timeout" 等模式
   - 重试模式（同一命令多次执行）
   - 恢复率（错误后被成功解决的比例）

4. **识别独特特性**（无法赛马的部分）：
   - 某些聊天在特定任务类型上的突出表现
   - 定性描述各聊天的独特优势

## 最小数据要求

- **最少 2 个聊天**（对比才有意义）
- **每个聊天至少 10 条消息**（统计才有意义）
- 如果不满足条件，发送简短通知："本周数据不足，跳过分析" 并退出

## 错误处理

1. 如果聊天记录文件读取失败，跳过该文件继续处理
2. 如果 `send_user_feedback` 失败，记录日志并重试一次
3. 如果没有任何有效数据，发送"数据不足"通知
4. 如果结构化日志不存在，仅基于聊天记录分析（降低精度但不阻塞）

## 注意事项

1. **零代码侵入**: 本任务仅分析现有数据，不修改任何代码
2. **AI 驱动分析**: 排名和评估由 AI 基于数据做出定性判断，不使用硬编码算法
3. **独特特性**: 始终包含"无法赛马"的部分，描述各框架的定性差异
4. **默认禁用**: `enabled: false`，需要手动启用
