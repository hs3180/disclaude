# Agent Framework 赛马周报

每周一 10:00 自动分析过去一周的聊天记录，对比不同模型/Provider 的服务质量表现。

## 执行步骤

### 1. 获取所有聊天记录文件

```bash
ls workspace/chat/*.md 2>/dev/null || echo "No chat files found"
```

如果 `workspace/chat/` 目录不存在或为空，跳过本次执行。

### 2. 分析聊天记录

对于每个聊天记录文件：

1. 读取文件内容：
   ```
   Read workspace/chat/{chatId}.md
   ```

2. 使用 `agent-race-review` skill 的分析框架，提取以下指标：

   - **响应效率**: 从消息时间戳推算响应速度
   - **任务完成度**: 分析对话轮次和最终结果（任务是否被完成标记）
   - **用户满意度**: 识别满意度信号（感谢、好评、差评、重复提问等）
   - **工具使用效率**: 统计工具调用次数与任务复杂度的关系
   - **错误率**: 统计任务失败、重试、超时等异常情况

3. 按模型/Provider 分组统计，生成对比数据

### 3. 生成赛马报告

按照 `agent-race-review` skill 的报告模板生成结构化报告，包含：

- 模型/Provider 总体表现对比表
- 各维度最佳表现者
- 详细分析（响应效率、任务完成度、错误模式、独特特性）
- 数据驱动的使用建议

### 4. 发送报告

使用 `send_user_feedback` 将报告发送到配置的 chatId：

```
send_user_feedback({
  chatId: "{chatId}",
  message: [生成的赛马报告],
  format: "text"
})
```

## 分析维度

| 维度 | 指标 | 权重 |
|------|------|------|
| 响应速度 | 首次响应时间、任务完成轮次 | 20% |
| 任务完成率 | 成功完成的任务比例 | 30% |
| 用户满意度 | 正面/负面反馈比例 | 25% |
| 错误率 | 失败任务比例 | 15% |
| 独特优势 | 创造性、主动性等定性指标 | 10% |

## 错误处理

1. 如果聊天记录文件读取失败，跳过该文件继续处理其他文件
2. 如果 `send_user_feedback` 失败，记录日志并重试一次
3. 如果分析数据不足（少于 3 个有效会话），在报告中注明数据局限性
4. 如果无法识别模型/Provider，按 chatId 维度进行分析

## 配置说明

使用前需要修改：
1. `chatId`: 替换为实际接收报告的聊天 ID
2. `enabled`: 设置为 `true` 启用
3. `cron`: 根据需要调整执行频率（默认每周一 10:00）
