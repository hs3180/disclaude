# Chats Cleanup

自动清理 `workspace/chats/` 中的孤儿 `.lock` 文件。

当 chat 文件被删除（如 chat-timeout 清理过期文件）但对应的 `.lock` 文件残留时，此 Schedule 负责清理这些孤立锁文件。

## 配置

- **Chat 目录**: `workspace/chats/`
- **执行间隔**: 每 30 分钟
- **每次最多处理**: 50 个（`CHAT_MAX_PER_RUN` 环境变量可覆盖）

## 职责边界

- ✅ 清理孤儿 `.lock` 文件（对应 `.json` 已不存在的锁）
- ✅ 清理过期 `.lock` 文件（对应 `.json` 存在但锁持有进程已死亡的锁）
- ❌ 不处理活跃的 `.lock` 文件（持有进程仍在运行）
- ❌ 不删除 `.json` 文件（由 `chat-timeout` skill 负责）
- ❌ 不创建或修改 chat 文件

## 执行步骤

```bash
npx tsx skills/chat/chats-cleanup.ts
```

### Step 2: 判断锁文件状态

1. **孤儿锁**: 对应的 `.json` 文件不存在 + 持有进程已死亡 → **删除**
2. **过期锁**: 对应的 `.json` 文件存在 + 持有进程已死亡 → **删除**
3. **活跃锁**: 持有进程仍在运行 → **跳过**

### Step 3: 安全删除

仅在确认锁持有进程已死亡（`process.kill(pid, 0)` 检测）后删除锁文件。

## 安全保证

| 保证 | 实现方式 |
|------|----------|
| **不误删活跃锁** | `process.kill(pid, 0)` 确认持有进程已死亡 |
| **并发安全** | `unlink` 是原子操作 |
| **有限处理** | 每次最多处理 50 个锁文件 |
| **无副作用** | 只删除 `.lock` 文件，不触碰 `.json` 文件 |

## 错误处理

| 场景 | 处理方式 |
|------|----------|
| Chat 目录不存在 | 正常退出（INFO: No chats directory found） |
| 锁文件在读取前被删除 | 跳过（文件已不存在） |
| 删除锁文件失败 | 记录警告，继续处理下一个 |
| 锁文件内容格式错误 | 视为无效锁，安全删除 |

## 注意事项

1. **轻量操作**: 不依赖 `lark-cli` 或其他外部工具
2. **低频执行**: 每 30 分钟执行一次，不会对系统造成负担
3. **防御性设计**: 宁可跳过可疑锁文件，也不误删活跃锁
4. **不创建新 Schedule**: 这是定时任务执行环境的规则
5. **不修改其他文件**: 只删除 `workspace/chats/` 目录下的 `.lock` 文件
