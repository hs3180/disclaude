# Chats Cleanup

自动清理 `workspace/chats/` 目录中的孤立 `.lock` 文件，保持目录整洁。

## 配置

- **Chat 目录**: `workspace/chats/`
- **执行间隔**: 每 1 小时
- **锁文件最大保留期**: 1 小时（`CHAT_LOCK_MAX_AGE_MS` 环境变量可覆盖，默认 3600000 毫秒）

## 前置依赖

- Node.js 20.12+（用于文件操作）

## 职责边界

- ✅ 清理孤立的 `.lock` 文件（对应 `.json` 文件已不存在）
- ✅ 清理过期的 `.lock` 文件（超过保留期且持有进程已死亡）
- ❌ 不处理 `workspace/chats/` 以外的文件
- ❌ 不删除 `.json` 文件（由 `chat-timeout` skill 负责）
- ❌ 不创建新 Schedule

## 执行步骤

```bash
npx tsx scripts/chats-cleanup.ts
```

脚本完整实现了以下逻辑：

### Step 1: 扫描 `.lock` 文件

遍历 `workspace/chats/*.lock`，查找所有锁文件。

### Step 2: 清理孤立锁文件

对每个 `.lock` 文件：
1. **检查对应 `.json` 文件是否存在** — 如果 `.json` 文件已被删除（chat 已清理），则 `.lock` 文件是孤立的
2. **检查锁文件年龄** — 如果 `.lock` 文件超过保留期（默认 1 小时）
3. **检查持有进程是否存活** — 解析 `.lock` 文件中的 PID，通过 `process.kill(pid, 0)` 检查
4. **安全删除** — 仅在确认孤立或过期时删除 `.lock` 文件

### 清理条件

| 条件 | 动作 |
|------|------|
| 对应 `.json` 文件不存在 | 删除 `.lock` 文件 |
| 锁文件超过保留期且持有进程已死 | 删除 `.lock` 文件 |
| 锁文件超过保留期且内容无效 | 删除 `.lock` 文件 |
| 锁文件未过期或持有进程存活 | 保留 `.lock` 文件 |

## 错误处理

| 场景 | 处理方式 |
|------|----------|
| `workspace/chats/` 目录不存在 | 正常退出（无需清理） |
| `.lock` 文件删除失败 | 记录警告，继续处理其他文件 |
| `.lock` 文件内容解析失败 | 视为无效锁文件，删除 |

## 注意事项

1. **安全清理**: 仅删除确认孤立或过期的 `.lock` 文件，不会误删活跃锁
2. **幂等性**: 重复执行不会产生副作用
3. **无状态**: 不维护内存状态，所有判断基于文件系统
4. **不创建新 Schedule**: 这是定时任务执行环境的规则
5. **不修改其他文件**: 只删除 `workspace/chats/` 目录下的 `.lock` 文件
